===== BEGIN src/theme/HowTo.txt =====
Here is what each theme parameter actually controls in your app, based on how it is wired through `ThemeContext`, Ant Design, and `CommonWidget.module.css`.

---

## 1. Top-level theme object fields

These live on each theme object (`lightTheme`, `darkTheme`, `cyberTheme`, etc.) and on the AntD-based ones in `src/theme/*.ts`.  

**Metadata**

* `label`
  Used only for display in the theme switcher button (`ThemeSwitcher`), e.g. ‚ÄúLight‚Äù, ‚ÄúDark‚Äù, ‚ÄúFunky Disco‚Äù.  
* `icon`
  Emoji/icon displayed next to the label in `ThemeSwitcher`.  

**Algorithm**

* `algorithm`
  Passed directly to `ConfigProvider.theme.algorithm`. Controls Ant Design‚Äôs base light/dark algorithm (how it derives full color palette and component styles from your seed tokens).  

---

## 2. Global Ant Design color tokens

These are read from the theme object (`raw`) and mapped into `tokenOverrides`, then given to `ConfigProvider.theme.token`. They influence every AntD component (buttons, inputs, cards, etc.). 

### Core semantic colors

* `colorPrimary` ‚Üí `token.colorPrimary`
  Primary brand / action color: primary buttons, selected controls, sliders, etc.  

* `colorPrimaryBg` ‚Üí `token.colorPrimaryBg`
  Soft background for primary elements (e.g. filled tags, subtle primary backgrounds). 

* `colorPrimaryHover` ‚Üí `token.colorPrimaryHover`
  Hover state of primary actions (buttons, links). 

* `colorPrimaryActive` ‚Üí `token.colorPrimaryActive`
  Active/pressed state of primary actions. 

* `colorInfo` ‚Üí `token.colorInfo`

* `colorSuccess` ‚Üí `token.colorSuccess`

* `colorWarning` ‚Üí `token.colorWarning`

* `colorError` ‚Üí `token.colorError`

These control colors for AntD `Alert`, validation states, status badges, etc. They are also used indirectly by `CommonWidget` semantic variants (see section 4).  

### Background colors

Mapped to AntD layout and container backgrounds: 

* `colorBgBase` ‚Üí `token.colorBgBase`
  Root background for body.
* `colorBgLayout` ‚Üí `token.colorBgLayout`
  Main layout background behind content.
* `colorBgContainer` ‚Üí `token.colorBgContainer`
  Default card/panel background.
* `colorBgElevated` ‚Üí `token.colorBgElevated`
  Elevated surfaces (modals, dropdowns, etc.).
* `colorBgMask` ‚Üí `token.colorBgMask`
  Overlay / modal backdrop.
* `colorBgSpotlight` ‚Üí `token.colorBgSpotlight`
  Used for spotlight/highlight surfaces.
* `colorBgSolid` ‚Üí `token.colorBgSolid`
* `colorBgSolidHover` ‚Üí `token.colorBgSolidHover`
  Solid-style backgrounds and their hover state.
* `colorBgPopconfirm` ‚Üí `token.colorBgPopconfirm` and also Popconfirm component background via `componentsOverrides.Popconfirm.colorBgElevated`.  

### Text colors

Mapped to AntD `token` and used by all typography in AntD components: 

* `colorText` ‚Üí `token.colorText` (default body text).
* `colorTextBase` ‚Üí `token.colorTextBase` (base neutral text).
* `colorTextSecondary` ‚Üí `token.colorTextSecondary` (muted descriptions).
* `colorTextTertiary` ‚Üí `token.colorTextTertiary` (subtle hints).
* `colorTextPlaceholder` ‚Üí `token.colorTextPlaceholder` (input placeholders).
* `colorTextDisabled` ‚Üí `token.colorTextDisabled` (disabled labels).

### Borders & separators

* `colorBorder` ‚Üí `token.colorBorder`
  Border color for inputs, cards, etc.
* `colorSplit` ‚Üí `token.colorSplit`
  Divider/line color between sections. 

### Shadows, radii, typography

* `boxShadow`, `boxShadowSecondary` ‚Üí `token.boxShadow`, `token.boxShadowSecondary`
  Default outer shadows for panels and secondary shadows. 

* `borderRadius` ‚Üí `token.borderRadius`

* `borderRadiusLG` ‚Üí `token.borderRadiusLG`
  Default corner rounding for most components and large ones. 

* `controlHeight` ‚Üí `token.controlHeight`
  Default vertical height for inputs/buttons. 

* `fontFamily` ‚Üí `token.fontFamily`

* `fontSize` ‚Üí `token.fontSize`

* `fontSizeLG` ‚Üí `token.fontSizeLG`
  Global typography family and base sizes used by AntD. 

* `paddingContentHorizontalLG`, `paddingContentVerticalLG` ‚Üí `token.paddingContentHorizontalLG`, `token.paddingContentVerticalLG`
  Default larger paddings for card content and similar containers. 

---

## 3. Component-specific theme parameters

These are mapped in `ThemeContext` into `theme.components` overrides. 

### Layout / Sider

* `layoutColorBgSider` ‚Üí `components.Layout.siderBg`
  Background color of the AntD `Layout.Sider` (side navigation).  

### Menu (side/top navigation menus)

The code normalizes the ‚Äúraw‚Äù keys into Menu tokens: 

* `colorMenuItemText` or `menuItemTextColor` ‚Üí `components.Menu.itemColor`
  Default text color of menu items.
* `colorMenuItemHoverBg` or `menuItemHoverBg` ‚Üí `components.Menu.itemHoverBg`
  Background when hovering a menu item.
* `colorMenuItemSelectedBg` or `menuItemSelectedBg` ‚Üí `components.Menu.itemSelectedBg`
  Background of the currently selected menu item.
* `colorMenuItemSelectedText` or `menuItemSelectedText` ‚Üí `components.Menu.itemSelectedColor`
  Text color for the selected menu item.
* `menuBg` ‚Üí `components.Menu.itemBg`
  Base background for the menu area. 

### Dropdown

* `dropdownBg` ‚Üí `components.Dropdown.colorBgElevated`
  Background of dropdown menus.
* `dropdownItemHoverBg` ‚Üí `components.Dropdown.controlItemBgHover`
  Background when hovering items in dropdown menus. 

### Popconfirm

* `colorBgPopconfirm` ‚Üí `components.Popconfirm.colorBgElevated`
  Background of Popconfirm popups. 

### Table

* `tableHeaderBg` ‚Üí `components.Table.headerBg`
  Background of AntD table headers. 

---

## 4. Custom CSS variables & `CommonWidget`

`ThemeContext` exports some theme parameters as CSS custom properties on `<html>`:  

* `bgMain` ‚Üí `--bg-main`
* `bgLight` ‚Üí `--bg-light`
* `bgDark` ‚Üí `--bg-dark`
* `textMain` ‚Üí `--text-main`
* `accent` ‚Üí `--accent`

These are then used (via fallbacks) by `CommonWidget.module.css` to derive widget styling:  

Internal widget variables (derived from theme):

* `--_bg`
  Base widget background; defaults to `--widget-bg`, `colorBgContainer`, or AntD container bg.
* `--_fg`
  Main widget text color; falls back to AntD `colorText`.
* `--_muted`
  Muted/secondary text (subtitles, footer).
* `--_border`
  Border color around the widget.
* `--_accent`
  Accent color used for focus rings, badges, and accent bar (`.widgetAccent`). If not explicitly set, falls back to `colorPrimary` / AntD primary.
* `--_title-fg`, `--_subtitle-fg`
  Colors for `widgetTitle` and `widgetSubtitle`.

Additional custom widget parameters:

* `--widget-bg`, `--widget-fg`, `--widget-muted`, `--widget-border`, `--widget-accent`, etc.
  Per-widget overrides (if set) that shadow the theme-level defaults.

Status-specific mapping:

* `.widgetSuccess` ‚Üí `--_accent` derived from `colorSuccess` or AntD success color.
* `.widgetWarning` ‚Üí from `colorWarning`.
* `.widgetError` ‚Üí from `colorError`.
* `.widgetInfo` ‚Üí from `colorInfo`. 

These determine the accent color for that widget‚Äôs border, skeleton, and focus ring.

### Cyber/dark theme CSS integration

The `<html>` element gets `data-theme={themeType}` and a `theme-{themeType}` class. 

`CommonWidget.module.css` then adds extra styling for:

* `[data-theme='dark']` / `.theme-dark`:
  Uses AntD `colorBgElevated` and `colorText`/`colorTextSecondary` to set a darker widget background, light text, and stronger border/shadow. 
* `[data-theme='cyber']` / `.theme-cyber`:
  Uses fixed neon values and `--cyber-*` variables (if provided) to create neon ‚Äúcyber‚Äù look (bg, fg, accent, border, glow). 

So in practice:

* **Theme selection (`themeType`)** controls which data-theme/class is applied.
* **Theme fields (`bgMain`, `textMain`, `accent`, etc.)** feed CSS variables consumed by widgets and any custom CSS using `var(--bg-main)` etc.
* **AntD color tokens (`color*`)** drive the default look of AntD components and also supply fallbacks and semantic colors for widgets.

---

## 5. Summary by parameter group

To recap, in this codebase:

* **`label`, `icon`** ‚Üí only affect the ThemeSwitcher button text/icon. 
* **`algorithm`** ‚Üí selects AntD‚Äôs light/dark algorithm. 
* **`color*` tokens** (primary/info/success/warning/error, backgrounds, text, borders) ‚Üí feed AntD `ConfigProvider.theme.token` and thereby control colors for all AntD UI plus widget status colors. 
* **`layoutColorBgSider`, `colorMenuItem*`, `menuBg`, `dropdownBg`, `dropdownItemHoverBg`, `tableHeaderBg`, `colorBgPopconfirm`** ‚Üí mapped into `components.Layout`, `components.Menu`, `components.Dropdown`, `components.Table`, `components.Popconfirm` to fine-tune sidebars, menus, dropdowns, tables, and popconfirms. 
* **`bgMain`, `bgLight`, `bgDark`, `textMain`, `accent`** ‚Üí exported as CSS vars on `<html>` and used by `CommonWidget` and any custom styles referencing `var(--bg-main)` etc.  

If you want, I can produce a compact table (parameter ‚Üí AntD token/CSS var ‚Üí visual effect) that you can paste into your theme docs.

export type ThemeId =
  | 'light'
  | 'modern'
  | 'dark'
  | 'ocean'
  | 'sunset'
  | 'cyber'
  | 'funky'
  | 'mauveRainbow';

export interface ThemeColors {
  // Brand + primary stack
  brand: string;
  primary: string;
  primarySoft: string;
  primarySubtle: string;

  // Surfaces
  background: string;
  backgroundAlt: string;
  surface: string;
  surfaceAlt: string;

  // Borders
  border: string;
  borderStrong: string;

  // Text
  text: string;
  textMuted: string;
